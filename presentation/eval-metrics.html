The 5 signals that feed them — the eval metrics

SignalSimple termsTechnical detail
CI status Did the build pass? Shipping on red is the worst thing you can do. required vs optional checks; merged-despite-failing & failing-required = critical, hard-block · pending · "no CI" = a gap
Merge speed Merged in minutes = nobody really looked. open→merge time: < 15 min = critical, < 60 min = high
Change size Huge diffs can't be reviewed well. additions+deletions ≥ 250 / 500 / 1000 = med / high / critical; ≥ 30 files flagged
Review quality Did a human approve it? 0 reviewers on a >10-line change = critical, hard-block; reviewed-but-not-approved = high; single reviewer = low
Ticket linkage Can we trace it to planned work? Jira key in title / branch / description (surfaced for traceability)
